# Mini NPU Simulator

## 1. 실행 방법
- **실행 명령어:** `python main.py`
- **모드 선택:**
  - `1` 입력: 사용자 입력 모드 (3x3 필터 및 패턴 직접 입력)
  - `2` 입력: JSON 데이터 분석 모드 (동봉된 `data.json` 파일 로드 및 일괄 판정)
- **주의사항:** 프로그램 실행 시 `data.json` 파일이 `main.py`와 동일한 디렉토리에 위치해야 합니다.

## 2. 구현 요약
- **라벨 정규화 방식:** 입력된 라벨의 공백을 제거하고 소문자로 변환(`strip().lower()`)한 뒤, `+`, `c`, `십자가` 등은 `Cross`로, `x`, `엑스` 등은 `X`로 통일했습니다. 알 수 없는 값은 `UNKNOWN`으로 처리하여 프로그램 오류를 방지했습니다.
- **MAC 연산 구현 개요:** 외부 라이브러리(NumPy 등)를 사용하지 않고, 2차원 리스트에 대해 이중 `for` 반복문을 사용하여 같은 위치의 인덱스 값을 곱(`Multiply`)하고 누적 합(`Accumulate`)하는 방식으로 직접 구현했습니다.
- **동점 처리 정책(epsilon):** 부동소수점 연산 시 발생할 수 있는 미세한 오차를 고려하여 허용오차(`epsilon = 1e-9`)를 설정했습니다. `abs(score_a - score_b) < 1e-9` 조건을 만족하면 두 점수가 같은 것으로 간주하여 `UNDECIDED` 판정을 내리도록 구현했습니다.

## 3. 결과 리포트

### 3.1. FAIL 케이스 원인 분석
JSON 분석 모드 실행 결과, 의도적으로 구성된 3가지 FAIL 케이스에 대해 프로그램이 중단되지 않고 정상적으로 예외를 처리했습니다.
1. **size_5_3_LogicalFail (정답 불일치):** 
   - 패턴은 명확한 'X' 형태를 띄어 MAC 연산 결과 `X`로 판정되었으나, JSON의 expected 값이 `+`로 잘못 기입되어 있어 로직 불일치로 FAIL 처리되었습니다.
2. **size_13_1_UndecidedFail (동점 처리 규칙):** 
   - 패턴에 십자가와 X 모양이 섞여 있어 두 필터의 MAC 연산 결과값이 완벽히 동일하게 나왔습니다. epsilon 규칙에 의해 `UNDECIDED`로 판정되었으며, 이는 expected 값과 다르므로 정상적으로 FAIL 처리되었습니다.
3. **size_13_2_SizeMismatchFail (데이터 스키마 불일치):** 
   - 13x13 필터 사이즈가 요구됨에도 불구하고 3x3 패턴이 입력된 케이스입니다. 연산 수행 전 `validate_size` 함수가 이를 미리 차단하여 IndexError를 방지하고 부드럽게 FAIL 처리 후 다음 로직으로 넘어갔습니다.

### 3.2. 성능 및 시간 복잡도 분석 (O(N²))
본 시뮬레이터의 MAC 연산은 외부 라이브러리의 벡터화 연산 없이 이중 반복문으로 구현되었습니다. 
- 패턴의 크기가 $N \times N$일 때, 행을 순회하는 외부 루프(N번)와 열을 순회하는 내부 루프(N번)가 중첩되므로 총 연산 횟수는 $N^2$번이 됩니다. 
- 따라서 이 알고리즘의 시간 복잡도는 **$O(N^2)$**입니다.
- 실제 성능 측정 표를 확인해보면 크기가 3x3, 5x5, 13x13, 25x25로 커짐에 따라 연산 횟수($N^2$)가 9, 25, 169, 625로 기하급수적으로 증가합니다.
- 연산 시간이 N의 제곱에 비례하여 증가하는 것을 측정된 평균 시간(ms) 데이터를 통해서도 직관적으로 확인할 수 있으며, 이는 고해상도 이미지 처리 시 기존 CPU의 직렬 처리 방식이 왜 병목을 일으키는지, 그리고 왜 병렬 처리에 특화된 NPU가 AI 연산에 필수적인지를 실증적으로 보여줍니다.
